# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm install        # Install dependencies
npm start          # Dev server with live reload (localhost:8080)
npm run build      # Production build ‚Üí _site/
npm run watch      # Watch mode without serving
npm test           # Run the Vitest test suite (one-shot)
npm run test:watch # Vitest in watch mode
```

**Always run `npm test` after making changes** ‚Äî the test suite catches broken builds, bad frontmatter, broken internal links, and data schema regressions.

## Architecture

This is an [Eleventy (11ty)](https://www.11ty.dev/) static site. `npm run build` reads `src/` and outputs plain HTML/CSS to `_site/`. Deployment happens automatically on push to `main` via GitHub Actions (`.github/workflows/deploy.yml`), which runs tests, builds, and FTP-deploys `_site/` to the hosting server. A separate CI workflow (`.github/workflows/ci.yml`) runs tests on all PRs and non-main pushes.

### CSS pipeline

CSS is **not** a simple passthrough copy. The build chain is:

```
src/_includes/css/tailwind.css  ‚Üí  PostCSS (autoprefixer + cssnano in prod)  ‚Üí  _site/css/styles.css
```

Tailwind scans `src/**/*.njk`, `src/**/*.md`, and `src/**/*.html` for class names. The `safelist` in `tailwind.config.js` keeps `header-anchor` (injected by the Markdown anchor plugin at build time, not present in source files).

To add a new utility class, use it directly in a template ‚Äî Tailwind picks it up automatically. To add a custom CSS rule, add it to `src/_includes/css/tailwind.css` inside the appropriate `@layer`.

### Template system

- **Layouts:** `src/_layouts/` ‚Äî `base.njk` wraps every page; `post.njk` extends it for blog posts.
- **Components:** `src/_includes/components/` ‚Äî partials included via `{% include %}`.
- **Pages:** `src/*.njk` ‚Äî each page declares `layout: base.njk` in frontmatter.
- **Blog posts:** `src/blog/*.md` ‚Äî Markdown with frontmatter. Collected into the `posts` collection in `eleventy.config.cjs`.
- **Tag pages:** `src/tags/` ‚Äî `index.njk` lists all tags; `tag.njk` paginates posts per tag. Slugs are generated by the `tagSlug` filter.

### Data layer

All structured content lives in `src/_data/` as JSON files, automatically available as template globals:

| File | Used by | Key schema |
|---|---|---|
| `site.json` | Every page (via `base.njk`) | `name`, `url` (https://), `email`, `social.{github,linkedin,handshake}` |
| `navigation.json` | `nav.njk` | Array of `{label, url}` ‚Äî `url` must be root-relative (start with `/`) |
| `projects.json` | Projects page & cards | See schema below |
| `experience.json` | About cards | `company`, `focusAreas`, `description` (required strings); optional `highlights` (array of non-empty strings) |
| `skills.json` | Skill groups | Array of `{heading, items: [{label, text}]}` |
| `tools.json` | Tools section | Array of `{name, category}` ‚Äî uppercase category strings |

**`projects.json` schema** (all fields required unless noted):

```json
{
  "title": "string",
  "description": "string",
  "image": "/images/...",
  "imageAlt": "string",
  "imageWidth": 800,
  "imageHeight": 600,
  "paragraphs": [{ "label": "string", "text": "string" }],
  "link": "https://..." ,   // optional ‚Äî null if no link
  "linkLabel": "string"     // required when link is non-null
}
```

### Eleventy configuration (`eleventy.config.cjs`)

The config file is `eleventy.config.cjs` (not `.eleventy.js`). The `.cjs` extension keeps CommonJS semantics without adding `"type": "commonjs"` to `package.json`, which would break `vitest.config.js` (ESM). The RSS plugin v3 is ESM-first; load its default export via `require("@11ty/eleventy-plugin-rss").default`.

Key behaviors defined here:

- **Plugins:** `@11ty/eleventy-plugin-rss@3` (Atom feed), `@11ty/eleventy-plugin-syntaxhighlight@5` (Prism code blocks).
- **Markdown:** `markdown-it` with `html: true`, `linkify: true`, `typographer: true`, and `markdown-it-anchor` for permalink anchors after headings. Slugify strips non-word chars and collapses separators.
- **Collections:**
  - `posts` ‚Äî all `src/blog/**/*.md` files.
  - `tagList` ‚Äî deduplicated, sorted array of every tag used across posts.
- **Filters** (implemented in `src/filters.js`):
  - `tagSlug` ‚Äî URL-safe tag slug (lowercase, hyphens, no punctuation).
  - `dateReadable` ‚Äî human-readable date string (`June 1, 2024`).
  - `dateIso` ‚Äî ISO 8601 datetime string; throws with `[dateIso]` prefix on invalid dates.
  - `dateYMD` ‚Äî `YYYY-MM-DD` string (used in sitemap).
  - `safeCdata` ‚Äî escapes `]]>` for valid CDATA in the Atom feed.
  - `readingTime` ‚Äî estimated read time at 200 wpm (minimum `"1 min read"`).
- **Global data:** `currentYear` (integer) and `buildDate` (`YYYY-MM-DD` string).
- **Passthrough:** `src/images/`, `src/files/`, `src/humans.txt`, `robots.txt`, `_headers`. (`src/.htaccess` is intentionally NOT passthrough ‚Äî Porkbun static hosting rejects it.)
- **Directories:** input `src/`, output `_site/`, includes `_includes/`, layouts `_layouts/`.

### Theme

Dark-first. `base.njk` sets `data-theme="dark"` on `<html>` via an inline `<script>` tag that runs before first paint. There is no runtime toggle ‚Äî dark is always the active theme. The `[data-theme="light"]` CSS rules in `tailwind.css` and `tailwind.config.js` exist for future opt-in support.

### Animations

Motion (from CDN: `https://cdn.jsdelivr.net/npm/motion@10.18.0/+esm`) drives three animation patterns in `base.njk`:

- **Hero stagger:** `data-motion="hero-item"` elements on `#hero` animate in with opacity + y-slide.
- **Section reveal:** `data-motion="section"` triggers on scroll into view.
- **Grid stagger:** `data-motion="stagger"` parent + `data-motion="stagger-item"` children.

All animation code is guarded by `prefers-reduced-motion` ‚Äî if the user prefers reduced motion, no animations run.

### Vue islands

Two pages use Vue 3 CDN islands for client-side interactivity:

- `src/projects.njk` ‚Äî project category filter
- `src/blog/index.njk` ‚Äî blog tag filter

Vue is loaded as an inline ES module import: `import { createApp } from 'https://cdn.jsdelivr.net/npm/vue@3.5.34/dist/vue.esm-browser.prod.js'`. The version is pinned to `3.5.34` ‚Äî do not change it to a floating `vue@3` range. SRI cannot be added to inline `import()` statements; the version pin is the supply-chain guard. When upgrading, bump both files together and run `npm test`.

### Deploy concurrency

`.github/workflows/deploy.yml` has a `concurrency: { group: ftp-deploy, cancel-in-progress: false }` block. This queues concurrent `main` pushes rather than running them in parallel, preventing interleaved FTP uploads that could corrupt the live host. Do not remove it.

## Test suite (`test/`)

Tests use [Vitest](https://vitest.dev/) and cover four areas:

| File | What it tests |
|---|---|
| `test/filters.test.js` | Unit tests for every function in `src/filters.js` |
| `test/data.test.js` | Schema validation for all `src/_data/*.json` files |
| `test/blog.test.js` | Frontmatter validation for every `src/blog/*.md` post |
| `test/build.test.js` | Full build smoke test: output files exist, CSS is minified (‚â§5 non-empty lines), passthrough artifacts present (`robots.txt`, `_headers`, `humans.txt`), internal links resolve, `<img>` tags have `alt`, OG/meta tags present, feed XML is valid |

**`test/build.test.js` runs `npm run build` in `beforeAll`.** It takes up to 2 minutes. If you add a new page or internal link, make sure the linked path exists in `_site/` after building.

## Blog post conventions

Every file in `src/blog/*.md` must have this frontmatter:

```yaml
---
title: "Post title"
description: "One-sentence summary shown in cards and meta tags."
date: 2024-06-01          # YYYY-MM-DD; dateIso will throw on invalid dates
tags: ["tag-one", "tag-two"]
layout: post.njk
thumbnail: /images/blog/your-image.jpg   # root-relative; file must exist
---
```

The `test/blog.test.js` suite enforces all of the above fields and verifies the thumbnail image file exists on disk.

## Design conventions

The site follows the design system defined in `DESIGN.md`. Key rules for AI assistants:

- **No emoji** anywhere ‚Äî not in copy, headings, or buttons.
- **No italics** ‚Äî use font weight (600/700) or uppercase mono labels for emphasis.
- **Single accent color** ‚Äî amber (`--accent: #f5a524`). No second accent, no gradients.
- **Sentence case** for headings and buttons; UPPERCASE only for mono section labels (`STACK`, `STATUS`).
- **Tailwind tokens** map to DESIGN.md tokens: `surface-{0-3,inset}`, `fg-{1-4}`, `hair-{1-3}`, `amber`, `font-{syne,plex,jb}`.
- **Dark surfaces** use `bg-surface-1` / `bg-surface-2` / `bg-surface-3` for depth, not shadows.
- **Breakpoints:** `sm: 481px`, `md: 801px`, `lg: 1101px` (overridden from Tailwind defaults).
- **Max widths:** `max-w-site` (1100px), `max-w-prose` (70ch).
- **Focus rings:** always visible ‚Äî `focus-visible:outline-[3px] focus-visible:outline-amber focus-visible:outline-offset-2`. Never `outline-none`.
- **Images** must always have `alt` attributes ‚Äî `test/build.test.js` enforces this.
